\subsection{The Quantum Threat}
As discussed in the introduction, the long-term security of modern cryptographic systems relies on the assumption that certain mathematical problems are computationally infeasible for classical computers to solve. The development of quantum computing challenges this assumption. Quantum computers operate according to the principles of quantum mechanics and can execute algorithms that outperform classical algorithms for specific problems. 

Two quantum algorithms are particularly important in this context: Shor's algorithm, which efficiently solves integer factorization and discrete logarithm problems, and Grover's algorithm, which accelerates brute-force search. Together, these algorithms form the foundation of the quantum threat to current cryptographic systems and are discussed in the following sections.

\subsubsection{Quantum Computing}
To understand how quantum computers can threaten modern cryptography, it is necessary to first give a high-level overview of how they operate.

A classical computer processes information using bits, which are binary values that exist in one of two states: 0 or 1. These bits are manipulated deterministically through electrical circuits consisting of transistors and logic gates. At any given time, a classical computer represents a single, well-defined state.

In contrast, a quantum computer operates on quantum bits, or \textit{qubits}. Unlike classical bits, qubits can exist in a superposition of both 0 and 1 at the same time. This means that a qubit represents a probability distribution over these states rather than a single fixed value. When multiple qubits are combined, the system can represent many possible states simultaneously. For a system with $n$ qubits, this corresponds to $2^n$ possible states, resulting in an exponentially large state space.

Another important property is entanglement. This occurs when the states of multiple qubits become correlated, such that the state of one qubit cannot be described independently of the others. Entanglement allows quantum computers to represent and exploit relationships between variables in ways that are not possible in classical systems.

Quantum algorithms make use of superposition, entanglement, and the manipulation of probability amplitudes. By carefully controlling how these amplitudes evolve during computation, quantum algorithms can increase the probability of correct solutions while reducing the probability of incorrect ones. This process enables quantum computers to achieve significant speedups for certain problems compared to classical computers \cite{Brassard2000Amplitude}.

\subsubsection{Shor's Algorithm}
One of the most important breakthroughs in quantum computing for cryptography was the discovery of Shor's algorithm by Peter Shor in 1994 \cite{shors}. This algorithm provides an efficient quantum method for solving integer factorization and the discrete logarithm problem, which are the foundation of many public-key cryptosystems.

To understand its impact, it is useful to consider the best known classical approach. The General Number Field Sieve (GNFS) is currently the most efficient classical algorithm for factoring large integers and runs in sub-exponential time \cite{GNFS}. Although it is much faster than naive methods, its computational cost still becomes infeasible for sufficiently large key sizes.

The largest RSA modulus factored using GNFS is RSA-250, which was completed in 2020 using the CADO-NFS implementation \cite{CADO-NFS}. This number is 829 bits long and required approximately 2700 CPU core-years of computation \cite{RSA250}.

Shor's algorithm fundamentally changes this situation. It can solve integer factorization and discrete logarithms in polynomial time. The algorithm works by reducing the problem to a period-finding problem. Given an integer $N$, it constructs a function whose period is related to the factors of $N$. While finding such periods is difficult for classical computers, quantum computers can do this efficiently using the Quantum Fourier Transform (QFT).

Later research has investigated the practical requirements for implementing Shor's algorithm. For example, Ekerå and Gidney showed in 2021 that factoring a 2048-bit RSA modulus could be possible using a quantum computer with approximately 20 million physical qubits and about 8 hours of runtime, assuming realistic error correction parameters \cite{Gidney2021RSA2048}.

Shor's algorithm therefore represents a serious threat to modern public-key cryptography. Protocols such as TLS, SSH, and VPN systems rely on RSA, Diffie-Hellman, and elliptic-curve cryptography for secure key exchange. If large-scale quantum computers become available, these systems would no longer provide security.

\subsubsection{Grover's Algorithm}
Grover's algorithm \cite{grovers} addresses a different type of problem than Shor's algorithm, namely unstructured search. For a search space of size $N$, a classical brute-force algorithm requires $\mathcal{O}(N)$ operations to find a target element. Grover's algorithm reduces this to $\mathcal{O}(\sqrt{N})$ using quantum amplitude amplification.

In cryptography, this affects symmetric-key algorithms such as block ciphers and cryptographic hash functions. Unlike Shor's algorithm, Grover's algorithm does not completely break these systems. Instead, it reduces their effective security level by a quadratic factor.

For example, a symmetric key of size $n$ bits provides $2^n$ security against classical attacks, but only $2^{n/2}$ security against a quantum attacker using Grover's algorithm. This means that a 128-bit key would offer approximately 64 bits of security in a quantum setting. To compensate for this reduction, it is generally recommended to double key sizes, for example by using 256-bit keys.

It should also be noted that Grover's algorithm has significant practical limitations, including large circuit depth and the need for error correction. These factors make its real-world implementation challenging in the near term. Nevertheless, it remains an important factor when evaluating long-term cryptographic security.

\subsubsection{``Harvest now, decrypt later.''}
A common misconception is that the quantum threat is not relevant until large-scale quantum computers exist. However, this ignores an important risk model known as ``harvest now, decrypt later'' \cite{hndl}.

In this scenario, an adversary collects and stores encrypted data today, with the intention of decrypting it in the future when quantum computers become sufficiently powerful. This is particularly problematic for information that must remain confidential for many years, such as government data, intellectual property, and personal records.

Public-key cryptography is especially vulnerable to this threat. Key exchange mechanisms such as RSA and Diffie-Hellman rely on problems that can be efficiently solved by Shor's algorithm. As a result, recorded encrypted communications could be decrypted retroactively once a suitable quantum computer is available.

This threat has important consequences for system design and cryptographic migration. It is not sufficient to consider only current security; future security must also be taken into account. This has increased the urgency of developing and deploying post-quantum cryptographic algorithms that are secure against both classical and quantum adversaries.

Therefore, the transition to quantum-resistant cryptography is not only a future concern, but also a necessary step to protect data that is being transmitted today.
